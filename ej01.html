    Preguntas sobre HTTP

  • 1) ¿Sobre qué protocolo binario está montado el protocolo HTTP?
  • Respuesta: El protocolo HTTP está montado sobre el protocolo TCP (Transmission Control Protocol).
  • 2) ¿Cuáles son los clientes HTTP y los servidores HTTP más utilizados?
  • Respuesta: Los clientes HTTP más utilizados incluyen navegadores web como Google Chrome, Mozilla Firefox, Microsoft Edge y Safari. Los servidores HTTP más comunes son Apache, Nginx, y Microsoft IIS.
  • 3) ¿Qué verbos admite un comando en el requerimiento HTTP?
  • Respuesta: Los verbos HTTP más comunes son GET, POST, PUT, DELETE, PATCH, HEAD, OPTIONS y CONNECT.
  • 4) ¿Qué contenido lleva el body de un requerimiento HTTP?
  • Respuesta: El cuerpo (body) de un requerimiento HTTP puede contener datos enviados al servidor, como formularios de usuario, archivos, o JSON. Este contenido depende del tipo de operación solicitada (como un POST o PUT).
  • 5) ¿Qué diferencia existe entre un URL y un URI?
  • Respuesta: Un URI (Uniform Resource Identifier) es un identificador genérico para un recurso en Internet, mientras que un URL (Uniform Resource Locator) es un tipo específico de URI que proporciona la dirección exacta para acceder a un recurso.
  • 6) ¿Cómo se almacena información relacionada con las respuestas HTTP?
  • Respuesta: La información relacionada con las respuestas HTTP se puede almacenar en caché del navegador, en cookies, o en bases de datos del lado del servidor.
  • 7) ¿Qué significa Virtual hosting?
  • Respuesta: Virtual hosting es una técnica utilizada en servidores web para alojar múltiples dominios (sitios web) en una sola dirección IP física.
  • 8) ¿Por qué el HTTP se considera STATELESS?
  • Respuesta: HTTP se considera stateless (sin estado) porque cada solicitud del cliente al servidor es independiente; el servidor no retiene información sobre solicitudes anteriores.
  • 9) ¿Qué nueva versión de HTTP se está usando para mejorar la velocidad de la WEB?
  • Respuesta: La versión más reciente de HTTP que se está utilizando para mejorar la velocidad de la web es HTTP/3, que se basa en el protocolo QUIC.
  • 10) ¿Cómo va a mejorar el comportamiento de una aplicación WEB en ambientes ruidosos o de débil conexión?
  • Respuesta: HTTP/3 mejora el comportamiento de las aplicaciones web en ambientes ruidosos o de débil conexión al usar el protocolo QUIC, que reduce la latencia y mejora la recuperación de paquetes perdidos.
